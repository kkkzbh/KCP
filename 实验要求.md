# 实验〇:实验工具及实验要求

### 0.1 实验工具

任何编程语言及编程工具均可作为编译程序的实现语言及工具,推 荐使用 C、C++、JAVA、C#、Python 等高级语言及可视化工具。

### 0.2 实验要求

(1)实验目标文法为 mini-C 语言文法、PL/0 语言文法或者 G[E]文 法之一即可。

### 文法:G[E]:

E→eBaA

A→a|bAcB

B→dEd|aC

C→e|dC

(2)实验采用多遍方法实现,即编译程序的每一部分都可以独立工 作,最终可以组装成一个完正的编译程序。

# 实验一:词法分析程序设计与实现

1.1 词法分析程序基本功能

目的:设计、编制、调试一个具体的词法分析程序,加深对词法分析原理 的理解。

## 要求:

- (1)通过对 PL/0 词法分析程序(GETSYM)的分析,认真研读描述 PL/0 语法规则,构造描述该语言词法的正规文法或正则表达式,将其转 换为不确定有穷自动机,将其确定化并最小化,在此基础上按照 PL/0 的 语法,编制一个词法分析程序。此程序应具有如下功能:
  - (a)输入为字符串(待进行词法分析的源程序),输出为单词串, 即由(单词,类别)所组成的二元组序列。
  - (b)有一定检查错误的能力,例如能发现 2A 这类不能作为单词 的字符串。
  - (2)提交实验报告,报告内容如下:

目的要求、分析说明、算法描述、程序结构、主要函数说明、主要变 量名说明、程序清单、调试情况、设计技巧、心得体会。

(3)上机时间:8 课时。

### 1.2 词法分析程序扩展功能

有时间的同学可以根据自己的实际情况在规定的课时内选择完成下 列扩展功能的部分或全部:

- (1)利用窗口编程机制,为词法分析程序设计并实现友好的人机界面。 该界面具备如下功能:
  - (a)具有编辑功能,即提供给用户一个编写源程序的编辑窗口。
  - (b)具有菜单功能,菜单中具备最基本的文件操作,即文件的"打 开"、"保存"功能以及对某一源程序进行词法分析的"分析" 功能。分析对象缺省为当前编辑窗口中的源程序,同时也支 持用户从任一路径指定特定的源文件(即浏览)功能。
  - (c)具有分析结果显示功能。以窗口的形式显示分析结果。
  - (2)具有不同单词类型不同颜色显示的彩色显示功能。
- (3)具有预处理功能。假设 PL/0 语言中的注释规则与 C 语言相同, 即在"/\*"和"\*/"中间的若干行符号或在"//"后面的一行符号为注释文字。 在进行词法分析之前,进行删除注释的预处理。
  - (4)根据同学学习进度可选做本实验。

# 实验五:语义分析及中间代码生成器设计与实现

- 目的:设计、编制、调试一个典型的编译器程序,实现对语法分析程序提 供的合法的语句进行语义分析、类型检查和中间代码生成,进一步 掌握常用的语义分析及中间代码生成方法。目的如下:
  - (1)加深对语法制导翻译原理的理解,掌握将语法分析所识别的语 法范畴变换为某种中间代码的语义翻译方法。
  - (2)掌握目前普遍采用的语义分析方法──语法制导翻译技术。
  - (3)给出目标文法规范,要求在语法分析程序中添加语义处理,对 于语法正确的表达式,输出其中间代码;对于语法正确的算术 表达式,输出其计算值。

### 要求:

- (1)在实验二或实验四的语法分析程序里,添加语义处理部分,输 出表达式的中间代码,用四元式序列表示。
- (2)语义分析对象重点考虑经过语法分析后已是正确的语法范畴, 本实验重点是语义子程序。
- (3)在实验二或实验四"语法分析器"的里面添加目标语言"表达式" 部分的语义处理,输出表达式的中间代码,计算表达式的语义 值。
- (4)中间代码用四元式序列表示。
- (5)上机时间:12 课时。
