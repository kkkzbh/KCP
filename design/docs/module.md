# 模块

模块用于组织源文件和控制文件之间的可见性。当前阶段不引入 namespace 概念，`import` 会把被导入模块导出的名字直接引入当前文件。

## 模块声明

```cp
export module math;
export module math.core;
```

一个文件最多声明一个模块。模块声明必须位于文件开头，并使用 `export module`。

没有模块声明的文件属于匿名模块。每个无模块声明文件都有独立的匿名模块，匿名模块不会按空名字合并，也不能被 `import` 引用。

当前允许多个文件声明同一个具名模块；这些文件会合并到同一个模块作用域中。

模块名由标识符组成，可以用 `.` 分隔:

```text
ModuleName -> identifier ( . identifier )*
```

`.` 只用于模块名，不表示表达式成员访问。

## 导入

```cp
import math;
import math.core;
```

导入语句位于模块声明之后、函数定义之前。

`import` 会把目标模块导出的名字引入当前文件作用域。使用导入的函数时直接写函数名:

```cp
import math;

main()
{
    let x = add(1, 2);
}
```

当前不支持选择性导入、别名导入、重导出，也不支持 `math::add` 这样的限定访问。后续如果需要名字限定，应单独设计 namespace 规则。

## 导出

具名模块内使用 `export` 修饰顶层函数，使其可被其他文件导入:

```cp
export module math;

export add(x: i32, y: i32) -> i32
{
    return x + y;
}
```

未标记 `export` 的顶层函数只在当前模块内部可见。匿名模块中的顶层函数不能使用 `export`，因为匿名模块没有可导入的模块名。

## 名字冲突

如果多个导入模块导出同名符号，或者导入符号与当前文件顶层定义同名，语义分析阶段应报告名字冲突。当前规则不使用限定名来消解冲突。
