# Lexer 正则表达式设计

本文档给出本语言词法分析阶段使用的 token 分类与对应正则表达式，用于实验一中“构造描述该语言词法的正规文法或正则表达式”的设计说明。

## 1. 设计原则

- 词法分析输入为源程序字符串，输出为 `(lexeme, token_kind)` 二元组序列。
- 词法分析采用最长匹配原则。
- 当同一词素既可匹配关键字又可匹配标识符时，关键字优先。
- 空白符与注释在词法阶段跳过，不进入语法分析。
- 标识符按当前语言设计限定为 ASCII，与 C++ 风格一致。

## 2. 基本字符类

为便于书写，先定义若干字符类记号：

- `LETTER = [A-Za-z_]`
- `DIGIT = [0-9]`
- `NONZERO = [1-9]`
- `IDENT_CONT = [A-Za-z0-9_]`
- `WS = [ \\t\\r\\n]+`

## 3. Token 分类与正则

### 3.1 空白符

- `WHITESPACE`
  - 正则：`[ \t\r\n]+`
  - 说明：匹配空格、制表符、回车、换行，扫描后直接跳过。

### 3.2 注释

- `LINE_COMMENT`
  - 正则：`//[^\n\r]*`
  - 说明：单行注释，从 `//` 开始到行尾结束。

- `BLOCK_COMMENT`
  - 正则：`/\*([^*]|\*+[^*/])*\*+/`
  - 说明：多行注释，风格与 C/C++ 一致。

## 3.3 标识符与关键字

### 标识符

- `IDENTIFIER`
  - 正则：`[A-Za-z_][A-Za-z0-9_]*`
  - 说明：仅支持 ASCII 标识符。

### 关键字

下列保留字在词法阶段识别为关键字，不作为普通标识符处理：

- `let`
- `const`
- `if`
- `else`
- `while`
- `do`
- `for`
- `break`
- `continue`
- `return`
- `import`
- `export`
- `module`
- `struct`
- `impl`
- `trait`
- `true`
- `false`
- `and`
- `or`
- `not`

说明：

- `true`、`false` 也可以看作布尔字面量，但在实现上通常作为关键字处理更直接。
- `and`、`or`、`not` 为逻辑运算符关键字，用于替代 C++ 中的 `&&`、`||`、`!`。

## 3.4 字面量

### 整数字面量

- `INTEGER_LITERAL`
  - 正则：`0|[1-9][0-9]*`
  - 说明：当前仅定义十进制整数。

### 浮点字面量

- `FLOAT_LITERAL`
  - 正则：`([0-9]+\.[0-9]+)([eE][+-]?[0-9]+)?|[0-9]+[eE][+-]?[0-9]+`
  - 说明：支持小数形式和科学计数法。

### 字符字面量

- `CHAR_LITERAL`
  - 正则：`'([^'\\]|\\.)'`
  - 说明：支持普通字符与转义字符，如 `'\n'`、`'\t'`、`'\\'`、`'\''`。

### 字符串字面量

- `STRING_LITERAL`
  - 正则：`"([^"\\]|\\.)*"`
  - 说明：支持转义字符，不允许未闭合字符串。

## 3.5 分隔符

以下符号可直接作为单字符或双字符 token：

- `LPAREN` : `\(`
- `RPAREN` : `\)`
- `LBRACE` : `\{`
- `RBRACE` : `\}`
- `LBRACKET` : `\[`
- `RBRACKET` : `\]`
- `COMMA` : `,`
- `SEMICOLON` : `;`
- `COLON` : `:`
- `DOT` : `\.`

### 作用域分隔符

- `SCOPE`
  - 正则：`::`
  - 说明：用于模块或命名空间访问，如 `math::add`。

### 返回类型箭头

- `ARROW`
  - 正则：`->`
  - 说明：用于函数返回类型声明。

## 3.6 运算符

本语言运算符优先级与 C++ 基本一致，但逻辑运算符使用 `and`、`or`、`not`。

### 算术运算符

- `PLUS` : `\+`
- `MINUS` : `-`
- `STAR` : `\*`
- `SLASH` : `/`
- `PERCENT` : `%`

### 赋值与比较

- `ASSIGN` : `=`
- `EQ` : `==`
- `NE` : `!=`
- `LT` : `<`
- `LE` : `<=`
- `GT` : `>`
- `GE` : `>=`

### 位运算与复合符号

- `AMP` : `&`
- `PIPE` : `\|`
- `CARET` : `\^`
- `TILDE` : `~`
- `LSHIFT` : `<<`
- `RSHIFT` : `>>`

### 自增自减

- `INC` : `\+\+`
- `DEC` : `--`

### 其他常见运算符

- `QUESTION` : `\?`

说明：

- `!` 在当前语言中不作为逻辑非运算符使用，逻辑非统一写作 `not`。
- 如果后续语法继续向 C++ 靠拢，可再补充 `+=`、`-=`、`*=`、`/=`、`%=`、`&=`、`|=`、`^=`、`<<=`、`>>=` 等复合赋值运算符。

## 4. 推荐匹配顺序

为避免短 token 抢占长 token，建议按下列顺序匹配：

1. 空白符与注释
2. 字符串字面量、字符字面量
3. 双字符及多字符运算符、分隔符
4. 浮点字面量
5. 整数字面量
6. 标识符或关键字
7. 单字符运算符、分隔符

其中第 3 步中应优先匹配：

- `::`
- `->`
- `==`
- `!=`
- `<=`
- `>=`
- `<<`
- `>>`
- `++`
- `--`

否则会出现 `:` 抢占 `::`、`-` 抢占 `->`、`<` 抢占 `<=` 等问题。

## 5. 错误词法单元

词法分析器应能识别并报告以下错误：

- 非法字符，如未定义用途的符号。
- 未闭合字符串，如 `"abc`
- 未闭合字符字面量，如 `'a`
- 未闭合块注释，如 `/* abc`
- 非法字符字面量，如 `''`、`'ab'`
- 数字与标识符直接相连形成非法单词，如 `2A`

对于 `2A` 这类输入，推荐报告为“非法数字后缀”或“数字与标识符未分隔”。

## 6. 词法输出示例

源程序：

```txt
export add(x: i32, const y: i32) -> i32 {
    return x + y;
}
```

可输出为：

```txt
("export", KEYWORD_EXPORT)
("add", IDENTIFIER)
("(", LPAREN)
("x", IDENTIFIER)
(":", COLON)
("i32", IDENTIFIER)
(",", COMMA)
("const", KEYWORD_CONST)
("y", IDENTIFIER)
(":", COLON)
("i32", IDENTIFIER)
(")", RPAREN)
("->", ARROW)
("i32", IDENTIFIER)
("{", LBRACE)
("return", KEYWORD_RETURN)
("x", IDENTIFIER)
("+", PLUS)
("y", IDENTIFIER)
(";", SEMICOLON)
("}", RBRACE)
```

## 7. 与自动机构造的关系

上述每一类 token 的正则表达式都可单独转换为对应的 NFA，再通过并联方式构造整个词法分析器的总 NFA，之后再进行 DFA 确定化与最小化。这与实验要求中的“正则表达式 -> NFA -> DFA -> 最小化 DFA”流程一致。

## 8. 后续实现建议

后续实现 lexer 时，建议采用如下策略：

1. 先做预处理或在扫描过程中直接跳过空白符和注释。
2. 维护关键字表，将标识符扫描结果再判定是否为关键字。
3. 按最长匹配原则识别多字符运算符。
4. 为每个 token 记录行号、列号，便于错误报告。
5. 对错误 token 继续恢复扫描，而不是直接终止，便于一次性发现多个问题。
